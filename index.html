Project Highlights

Legal-Tech Production System

A BRAO and GDPR-compliant web app for a German law firm, taken from an inherited MVP to production. (Client work, confidential, so this is the architecture rather than screenshots.)

A self-hosted n8n layer syncs the app with the firm's external case-management and accounting system both ways, over REST and webhooks. Every AI call goes through one EU-only anonymisation gateway with a sub-processor allowlist, so nothing identifiable ever leaves the approved providers. It also handles document generation with Gotenberg, a state machine for court deadlines, and an in-app agent for searching cases.

Next.js 15 n8n Supabase Azure OpenAI EU Gotenberg BRAO / GDPR

Career Intelligence SaaS

A three-repo subscription platform I built solo, end to end: a Flask backend, a bilingual Next.js frontend, and a standalone headless aggregator, each with its own CI/CD.

The core is a five-stage evaluation pipeline that turns raw listings into scored matches. It runs SQL constraints first, then skill-overlap, then batched LLM scoring across parallel workers, then a re-rank on the top group. Model routing is cost-tiered across Anthropic and OpenRouter, output is JSON-schema enforced, every operation has a fallback chain, and billing runs on credits with atomic RPC deductions and Stripe webhooks.

Next.js 15 Flask PostgreSQL Supabase Multi-Model AI Stripe

Data Aggregation Pipeline

A headless ingestion engine that runs unattended across 29 different sources.

The 29 parsers cover REST APIs, ATS systems, RSS feeds, and HTML scrapers, and pull tens of thousands of records per run into one canonical store. Deduplication runs in three layers (external ID, strict hash, fuzzy hash), with a batch pre-fetch that collapses 3N lookups down to three queries. Each source has its own circuit breaker that auto-resets after 24 hours, and adding a new source means one parser file and a config entry, nothing else.

Python PostgreSQL BeautifulSoup Docker Resilience Engineering

Multi-Tenant Field Sales PWA

A mobile-first B2B tool that replaces Excel for FMCG field reps during retail campaigns.

Every table has row-level security through a SECURITY DEFINER membership function, and organisations provision themselves on first login. There is a campaign, category, and variant hierarchy with template cloning, four quantity modes for order entry, and idempotent upserts under a unique constraint. The PWA tolerates a flaky connection and is built for reps over fifty: big touch targets, as little typing per order as possible.

Next.js 15 TypeScript Supabase Tailwind Row-Level Security

Solidity Security Scanner

Static analysis plus an LLM deep-review pass for EVM smart contracts, packaged as a subscription product.

It checks ten vulnerability classes (reentrancy, access control, integer overflow, unchecked calls, delegatecall, and more) by pairing Slither with a Claude review pass. It pulls verified source from eight EVM chains through Etherscan, offers three scan depths by tier, and follows a no-exploit-no-report rule to keep false positives down. 154 tests back it.

Python FastAPI Slither Anthropic Next.js Stripe
